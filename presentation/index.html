MVC

  • M is for Model
  • V is for View
  • C is for COLLECTION!

There are Controllers, But in this case, we are simplifying things by just talking about interactions between a collection of models and their views via custom and DOM events

Model

Representation of Data

  • Is reusable
  • Can be persisted
  • Can throw custom events

Collection

A collection of... MODELS!

  • Great for performing manipulations and computations on multiple models
  • Can be persisted
  • A great listener!
    • Can listen to attribute changes on it's models
    • Can listen to custom events thrown by models

View

Links the DOM and the models

  • Manipulate the DOM based on model state and attributes
  • Updates models based on user interaction with the DOM
  • It's all about events!

Models

                            
App.Todo = Backbone.Model.extend({

    defaults: {
        title: '',
        complete: false
    },

    doSomething: function () {
        // I should really write a todo to finish this function...
    }

});

Some useful functions

                            
// Getting attributes within model.
Model.get('complete');


// Setting attributes within model.
Model.set('complete', true);


// Saving attributes to the synced database.
Model.save({
    complete: true
});

Collections

                            
App.TodoCollection = Backbone.Collection.extend({

    // Reference to the model of this collection.
    model: App.Todo

});

Some useful functions

                            
// Fetching models from synced database.
Collection.fetch({
    success: function (collectionOfThings) {
        // Should really do something with these things...
    }
});


// Creating and adding new model to the collection.
Collection.create({
    title: 'Create Todo List'
});

Views

                            
App.TodoView = Backbone.View.extend({

    initialize: function() {
        // First function that is called when you create a new view.
        // Great for setting up event listeners.
    },

    // DOM events mapping to functions within this view.
    events: {
        'click .toggle-complete': 'toggleComplete'
    },

    toggleComplete: function() {
        // OK I'm done!
    }

});

Some useful functions

                            
// Access the views model
View.model;


// Access the views collection
View.collection;


// Listen to model attribute change events
View.listenTo(this.model, 'change', this.render);
